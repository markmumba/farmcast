# FarmCast AI

FarmCast AI is a full-stack WeatherAI demo for farm forecasting and crop intelligence. The Angular frontend gives farmers a clean dashboard for weather, daily outlooks, hourly conditions, API usage, and tree image analysis. The Spring Boot backend proxies WeatherAI requests so the API key stays server-side.

## Tech Stack

- Frontend: Angular, PrimeNG, SCSS
- Backend: Spring Boot, Java 17, Maven
- Deployment: Render static site plus Docker web service

## Project Structure

```text
.
├── backend/       # Spring Boot API adapter
├── frontend/      # Angular + PrimeNG application
├── render.yaml    # Render blueprint
└── .node-version  # Node version for Render/frontend builds
```

## Prerequisites

- Java 17
- Maven
- Node 22.12.0 or newer compatible Angular 21 runtime
- npm
- WeatherAI API key

## Backend Setup

Create the backend env file:

```bash
cd backend
cp .env.example .env
```

Update `backend/.env`:

```text
SERVER_PORT=8080
FRONTEND_URL=http://localhost:4200
WEATHER_AI_BASE_URL=https://api.weather-ai.co
WEATHER_AI_API_KEY=your_weather_ai_key_here
```

Run the backend:

```bash
mvn spring-boot:run
```

The backend starts at:

```text
http://localhost:8080
```

Health check:

```text
GET http://localhost:8080/api/health
```

## Frontend Setup

Create the frontend env file:

```bash
cd frontend
cp .env.example .env
```

Update `frontend/.env`:

```text
BACKEND_URL=http://localhost:8080
```

Install dependencies:

```bash
npm install
```

Run the frontend:

```bash
npm start
```

The frontend starts at:

```text
http://localhost:4200
```

## Render Deployment

This repo is set up for two Render services:

- Backend: Docker web service
- Frontend: Static site

You can use the root `render.yaml` blueprint, or create the services manually.

Backend settings:

```text
Runtime: Docker
Dockerfile path: backend/Dockerfile
Docker context: backend
Health check path: /api/health
```

Backend environment variables:

```text
WEATHER_AI_BASE_URL=https://api.weather-ai.co
WEATHER_AI_API_KEY=your_weather_ai_key_here
FRONTEND_URL=https://your-frontend-service.onrender.com
```

Frontend settings:

```text
Runtime: Static Site
Build command: cd frontend && npm install && npm run env && ./node_modules/.bin/ng build
Publish directory: frontend/dist/farmcast-ai/browser
```

Frontend environment variables:

```text
NODE_VERSION=22.12.0
BACKEND_URL=https://your-backend-service.onrender.com
```

Frontend rewrite rule:

```text
Source: /*
Destination: /index.html
```

After the first Render deploy, confirm the actual Render URLs. If Render assigns different URLs from the blueprint defaults, update `FRONTEND_URL` on the backend and `BACKEND_URL` on the frontend, then redeploy both services.

## Notes

- Do not commit `.env` files or the WeatherAI API key.
- The frontend reads `BACKEND_URL` through `frontend/public/env.js`, generated by `npm run env`.
- The backend reads Render's `PORT` automatically and falls back to `SERVER_PORT` or `8080` locally.
